 Fichiers de contrôle

 Le comportement de démarrage de Notepad++ peut être ajusté avec la création de fichiers vides (0 octet):
 doLocalConf.xml
 S'il est présent, Notepad++ utilise le répertoire d'installation pour toutes sa configurations. S'il est absent, c'est %APPDATA% qui est utilisé à la place. Ce choix est forcé sous Windows Vista et suivants, hors du mode Administrateur. Les plugins sont toujours dans un sous-répertoire du répertoire d'installation.
 asNotepad.xml
 S'il est présent, Notepad++ imitera le Bloc-notes de  Windows, ce qui signifie qu'il sera lancé dans plusieurs instances, la barre d'onglets non visible et sans charger la session précédente.
 allowAppDataPlugins.xml
Ce fihchier ne doit être présent que si vous autorisez l'installation de plugins dans %APPDATA%\Notepad++. Le gestionnaire Plugin Manager peut devoir le faire sous Windows Vista ou plus, en fonction de la conifiguration de l'UAC.
