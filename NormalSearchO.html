RechercherSuivant: Remplacer

 Recherche normale


    Vous pouvez rechercher du texte comme dans la plupart des programmes utilisant la boîte de dialogue Rechercher / Remplacer. Vous pouvez ouvrir cette boîte de dialogue par:
  • Rechercher ->  Rechercher (ou en appuyant sur  ),
  • Rechercher -> Remplacer (ou en appuyant sur  )
  • Ou Recherche-> Rechercher dans les fichiers.

 Dans chacune de ces options, ouvrez l'onglet correspondant dans le dialogue Rechercher / Remplacer. Cette boîte de dialogue restera toujours au-dessus du texte, mais vous pouvez toujours continuer à modifier le texte si lle est ouvert, et vous pouvez la configurer pour qu'elle soit transparente (en utilisant l'option Transparence), lors de son inactivité (par perte du focus) ou tout le temps. Le curseur contrôle le niveau de transparence. Notez que la transparence n'est disponible quà partir de Windows 2000.

 Dans le champ Rechercher, tapez le texte que vous souhaitez trouver. Cette zone est automatiquement remplie avec le texte sélectionné en cours, ou le mot sous le curseur, ou le dernier modèle de recherche, lorsque la boîte de dialogue Rechercher est ouverte. La liste déroulante contient des éléments de recherche antérieure.

     Les options suivantes vont limiter la recherche:
  •   Mot entier uniquement  : la recherche n'aboutit que si le texte recherché constitue un mot complet (par exemple : si la chaîne trouvée est bornée par des blancs ou des caractères de ponctuation).
  •   Respecter la casse : la chaine recherchée le sera à la majuscule /minuscule près. Si cette option est désactivée, la recherche est insensible à la casse.
  •   Boucler  continue en haut du fichier lorsque la recherche rencontre la fin de fichier, quel que soit l'emplacement initial du curseur.
     Le mode de recherche détermine la façon dont la requête sera traitée :
  •  En Mode Normal , le texte sera recherché littéralement.
  •  En Mode étendu , des séquences de caractères spéciaux seront interprétées. Dans ce cas, vous pouvez choisir entre :
    •  \\ - Barre oblique inverse;
    •  \t - caractère de tabulation;
    •  \r - caractère CR;
    •  \n - caractère LF;
    •  \0 - caractère NULL;
    •  \x## - Valeur hexadécimale (entre 00 et FF);
    •  \u#### - valeur hexadécimale étendue entre 0000 et FFFF, signifiait pour les caractères Unicode);
    •  \d### - valeur Décimal (entre 000 et 255);
    •  \o### - valeur octal (entre 000 et 377);
    •  \b######## - Valeur binaire (entre 00000000 et 11111111).
  •  Dans  une expression régulière , la requête sera traitée comme une expression régulière. Notepad++ utilise la syntaxe PDRE, qui est un standard bien établi. Vous trouverez ici une documentation complète. Cette page fournit quelques exemples simples (ces deux documents sont en anglais).

Dans la liste ci-dessus, # signifie «un chiffre». Selon le cas, «chiffre» peut avoir un sens différent: par exemple, en binaire, 0 et 1 sont les seuls chiffres.

 Le sens de la recherche détermine dans quelle direction, par rapport au curseur, la recherche sera effectuée. Haut signifie que c'est le texte avant le curseur qui sera analysé, bas signifie que c'est le texte après le curseur qui sera analysé.

Normalement, lors d'un Find, le texte trouvé est sélectionné. Une difficulté, n mode Expressions rrégulières, est que le texte trouvé peut être de longueur nulle, par exemple quand on recherche "^". Dans ce cas, une infobule montr la position de la chaîne vide résultat.

 Pour trouver l'occurrence suivante dans le sens de la recherche, appuyez sur Suivant (ou Enter). Compter compte le nombre d'occurrences de la recherche, Trouver tous dans tous les documents ouverts   va lancer la recherche sur tous les documents ouverts et affiche une liste de résultats pour chaque fichier, voir aussi la recherche dans les fichiers. Trouver tous dans les documents actuels se comporte d'une manière très similaire.

 Fermer ferme la boîte de dialogue de recherche (bouton de fermeture et l' Echap font la même chose).

Localisation de caractères

Une des utilisations possibles de la fonction de recherche est de repérer les occurrences de certains types de caractères dans un document, par exemple des letres accentuées qu'il vous faut convertir en entités HTML. Ceci peut se faire via la boîte de dialogue Rechercher -> Trouver des caractères d'une plage.... Ce dialogue vous permet de Trouver le prochain ou précédent cractère faisant partie de la plage ASCII (0..127), non ASCII (128..255) ou d'une dont vous spécifiez les limites - les valeurs supérieures à 255 ne sont pas correctement interprétées. La recherche peut Boucler à votre demande.
