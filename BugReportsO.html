 Rapports de bugs

 Si vous trouvez que Notepad++ ne fonctionne pas correctement ou tout simplement s'il se fige, vous avez probablement rencontré un bug.  Vérifiez d'abord si vous avez la dernière version, chaque édition de Notepad++ est améliorée et les bugs sont corrigés (malheureusement, on en ajoute parfois, si bien que la bonne démarche est toujours d'être prudent lors d'une la mise à jour).  En outre, si vous venez d'ajouter ou upgrader un plugin, il pourrait être la cause du bug.

     Ensuite, vérifiez si un plugin cause des problèmes.  Supprimer tous les plugins ou démarrer avec l'option de ligne de commande pour désactiver tous les plugins et regardez si le problème persiste.  Si c'est le cas:
  •  Vérifiez les forums: peut-être d'autres utilisateurs ont rencontré ce problème et trouvé une solution.
  •  Vérifiez le "bug tracker" ( http://sourceforge.net/tracker2/?group_id=95717&atid=612382 ): le bug peut déjà être détecté et dans la liste. Recherchez s'il fait partie de la liste, sinon, vous pouvez l'ajouter et aider les développeurs à le diagnostiquer et à le corriger (de manière la plus claire possible : quel est le problème? Quelles sont ses causes et comment le reproduire? Les rapports d'erreur sont, malheureusement, actuellement très peu utiles, une description sur la façon de reproduire le crash est indispensable. Le cas échéant, essayez également d'envoyer en zip ou 7-zip le fichier NppDump.dmp s'il a été produit: ce rapport contient l'état interne de Notepad++ lorsque l'accident a eu lieu, un trésor inestimable pour les développeurs.
